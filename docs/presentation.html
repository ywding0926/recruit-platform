3
Core Feature

官网 × 招聘系统打通机制

双通道设计：定时同步 + 实时 Webhook，确保候选人零遗漏、零延迟地进入招聘流水线。

⏱️ 通道一：定时同步（Pull 模式）

  • 每 15 分钟 自动执行一次
  • 从 Careers 管理后台拉取全量投递数据
  • 三重去重：careersAppId → 姓名+手机 → 姓名+邮箱
  • 自动匹配本地职位（关键词映射表）
  • 未匹配的职位自动创建
  • 下载简历并存储到 Supabase
  • 通过飞书通知 HR 有新候选人
  • 管理员可在设置页手动触发

⚡ 通道二：Webhook（Push 模式）

  • 候选人投递后 实时 推送到系统
  • Careers 官网提交表单后自动触发
  • 通过 X-Webhook-Secret 密钥验证安全性
  • 同样执行三重去重逻辑
  • 同样自动匹配/创建职位
  • 异步下载简历（不阻塞响应）
  • 飞书实时通知 HR
  • 适合 Vercel 无服务器环境

💡 为什么用双通道？

Webhook 保证实时性——候选人投递后几秒内 HR 就能看到。定时同步 作为兜底——即使 Webhook 偶尔失败（网络问题/服务重启），15分钟后也会自动补齐，确保不会遗漏任何候选人。两者配合，实现 零丢失 的数据同步。

4
Deep Dive

定时同步技术流程

从登录认证到候选人入库，完整的 7 步同步流水线。

1

认证登录 Careers 管理后台

系统自动使用预设的账号密码，通过 POST /api/login 获取 Session Cookie，建立安全连接。

2

拉取官网职位列表

调用 GET /api/jobs 获取 Careers 官网所有在招职位，提取部门、地点等信息用于后续匹配。

3

拉取全量投递数据

调用 GET /api/admin/applications 获取所有候选人投递记录，包含姓名、联系方式、应聘岗位等。

4

三重去重检查

依次检查：① careersAppId 是否已存在 → ② 姓名+手机是否匹配 → ③ 姓名+邮箱是否匹配。已存在则跳过，避免重复导入。

5

智能职位映射

通过关键词映射表将官网职位名称匹配到系统内部职位。例如 "AI product manager" → "AI产品经理"。无法匹配时自动创建新职位。

6

创建候选人 + 下载简历

在系统中创建候选人记录（状态：待筛选，来源：官网投递），同时从管理后台下载简历 PDF 存入 Supabase 云存储。

7

飞书通知 HR

通过飞书 API 向对应岗位负责人（或全体管理员）发送消息，包含候选人信息和系统链接，HR 可立即开始处理。
