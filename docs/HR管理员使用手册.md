# MachinePulse 招聘管理系统 — HR / 管理员使用手册

---

## 一、系统概述

MachinePulse 招聘管理系统是一套面向中小团队的一站式招聘协作平台，涵盖候选人管理、面试安排、Offer 管理、猎头协作、官网职位同步等完整招聘流程。系统通过飞书（Lark）实现单点登录、消息通知、日历同步与任务协同。

**管理员（admin）** 拥有系统全部权限，可管理所有候选人、职位、团队成员、系统设置等。

## 二、登录与认证

### 2.1 飞书扫码登录
- 访问系统首页，点击 **"飞书登录"** 按钮
- 跳转至飞书授权页面，使用飞书 App 扫码确认
- 授权完成后自动跳转回系统首页
- 系统会自动匹配飞书账号与系统用户，如果是首次登录且用户列表中有对应名称的用户，会自动关联

### 2.2 账号密码登录
- 管理员也可通过用户名+密码方式登录
- 用户名和密码由管理员在「设置」页面创建用户时指定

## 三、首页仪表盘

登录后进入首页仪表盘，包含以下信息：

### 3.1 统计卡片
- **候选人总数**：系统中所有候选人的数量
- **进行中**：当前处于流程中（非"入职"/"淘汰"）的候选人数量
- **本月新增**：本月新录入的候选人数量
- **在招职位数**：当前状态为"招聘中"的职位数量

### 3.2 待办提醒
- **今日面试**：显示今天有面试安排的候选人列表，包含面试轮次、时间和面试官信息，点击可跳转至候选人详情
- **待面评**：显示已面试但面试官尚未提交评价的候选人，包含轮次和面试官，点击可跳转

## 四、候选人管理

### 4.1 候选人列表

路径：顶部导航 → **候选人**

列表展示所有候选人的基本信息，支持以下操作：

#### 搜索与筛选
- **搜索框**：输入姓名、手机号或备注关键词，支持实时搜索（输入即搜，400ms 防抖）
- **岗位筛选**：下拉选择特定岗位，切换后立即筛选
- **来源筛选**：下拉选择候选人来源，切换后立即筛选

#### 列表字段
| 字段 | 说明 |
|------|------|
| 姓名 | 候选人姓名，点击进入详情页 |
| 应聘岗位 | 关联的职位名称 |
| 状态 | 当前所处的招聘阶段（彩色标签） |
| 来源 | 候选人来源渠道 |
| 标签 | 自定义标签（如"高潜"、"紧急"等） |
| 创建时间 | 录入系统的时间 |

#### 操作按钮（管理员可见）
- **+ 新增候选人**：手动创建候选人
- **导入 Excel**：批量导入候选人（支持 .xlsx 文件）
- **导出 Excel**：将当前筛选结果导出为 Excel 文件

### 4.2 新增候选人

点击 **"+ 新增候选人"** 按钮，弹出表单：

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 姓名 | 是 | 候选人姓名 |
| 手机 | 否 | 联系电话 |
| 邮箱 | 否 | 电子邮箱 |
| 应聘岗位 | 否 | 下拉选择已有职位 |
| 来源 | 否 | 下拉选择来源渠道 |
| 简历附件 | 否 | 上传简历文件（PDF/Word 等，最大 20MB） |
| 备注 | 否 | 自由文本备注 |

提交后系统自动将候选人状态设为 **"待筛选"**，并记录创建事件到时间线。

### 4.3 批量导入

点击 **"导入 Excel"** 按钮，上传包含候选人信息的 .xlsx 文件。

Excel 文件要求：
- 表头行必须包含字段名（姓名、手机、邮箱、应聘岗位、来源、备注等）
- 系统会自动匹配列名
- 导入后每位候选人初始状态为"待筛选"
- 如果 Excel 中的"应聘岗位"在系统中不存在，系统会自动创建该职位

### 4.4 候选人详情页

点击候选人姓名进入详情页，包含以下区域：

#### 基本信息区
- 姓名、手机、邮箱、应聘岗位、来源、标签
- 当前状态（可通过下拉框修改）
- 操作按钮：编辑信息、删除候选人

#### 简历区
- 查看已上传简历（支持在线预览）
- 上传/替换简历文件
- 简历存储支持 Supabase 云存储 或 本地存储

#### 面试安排区
- 显示所有已安排的面试轮次
- 每轮面试信息：轮次编号、面试官、面试时间、面试地点/链接、飞书日历同步状态
- **+ 安排面试**：新增面试轮次

#### 面试评价区
- 显示每轮面试的评价结果
- 评分等级：S / A / B+ / B / B- / C
- 评价内容、优势、不足、面试官签名
- 系统会根据面评自动推进候选人状态（如通过→进入下一轮面试）

#### 时间线
- 记录候选人的所有状态变更、操作事件
- 包含：创建、状态变更、面试安排、面试评价、备注添加等
- 每条记录包含时间戳和操作人

#### 备注区
- 添加自由文本备注
- 历史备注按时间倒序展示

### 4.5 安排面试

在候选人详情页点击 **"+ 安排面试"**：

| 字段 | 说明 |
|------|------|
| 面试轮次 | 第 1/2/3/4/5 轮 |
| 面试官 | 下拉选择系统成员 |
| 面试时间 | 日期 + 时间选择器 |
| 面试时长 | 默认 60 分钟 |
| 面试地点/链接 | 文本输入（如会议室名称或视频链接） |
| 同步飞书日历 | 勾选后自动在飞书创建日历事件 |

安排成功后：
- 面试信息显示在候选人详情页
- 如勾选飞书日历同步，会自动在面试官和候选人的飞书日历中创建事件
- 系统向面试官发送飞书通知
- 面试安排记录到候选人时间线

### 4.6 修改候选人状态

两种方式修改状态：

1. **详情页下拉框**：在候选人详情页的状态下拉框中选择新状态
2. **看板拖拽**：在看板视图中将卡片拖拽到目标列
3. **快捷按钮**：在看板卡片上hover显示"推进 →"和"淘汰"按钮

状态流转路径（16 个阶段）：
```
待筛选 → 简历初筛 → 待一面 → 一面通过 → 待二面 → 二面通过 →
待三面 → 三面通过 → 待四面 → 四面通过 → 待五面 → 五面通过 →
待发offer → Offer发放 → 入职
                                         ↘ 淘汰（任意阶段可淘汰）
```

## 五、看板视图

路径：顶部导航 → **看板**

看板提供候选人状态的可视化管理，支持两种视图模式：

### 5.1 流水线视图（默认）
将 16 个状态聚合为 6 个阶段列：
| 阶段 | 包含的状态 |
|------|-----------|
| 简历筛选 | 待筛选、简历初筛 |
| 面试中 | 待一面、一面通过、待二面、二面通过、...、五面通过 |
| 待发Offer | 待发offer |
| Offer已发 | Offer发放 |
| 已入职 | 入职 |
| 淘汰 | 淘汰 |

### 5.2 状态视图
每个状态一列，共 16 列，可横向滚动。适合需要精细查看每个状态的场景。

### 5.3 看板操作
- **点击卡片**：跳转到候选人详情页
- **快捷按钮**：鼠标悬停在卡片上，显示"推进 →"（推进到下一状态）和"淘汰"按钮
- **岗位筛选**：顶部下拉框可按岗位过滤看板内容
- **视图切换**：点击"流水线视图"/"状态视图"按钮切换

## 六、职位管理

路径：顶部导航 → **职位**

### 6.1 职位列表
展示所有职位信息：
- 职位名称、部门/类别、状态（招聘中/已暂停/已关闭）
- 关联的候选人数量
- 创建时间

### 6.2 新增职位

| 字段 | 说明 |
|------|------|
| 职位名称 | 必填 |
| 部门/类别 | 下拉选择（技术/产品/设计/运营/市场/销售/人力/财务/行政/其他）|
| 职位描述 | 富文本描述 |
| 职位要求 | 任职要求描述 |
| 工作地点 | 文本输入 |
| 薪资范围 | 文本输入 |
| 职位状态 | 招聘中 / 已暂停 / 已关闭 |

### 6.3 官网职位同步（Careers Sync）

系统支持将职位自动同步到公司官网的 Careers 页面：
- **自动同步**：每 15 分钟自动执行一次
- **Webhook 触发**：外部系统可通过 Webhook 触发同步
- **手动触发**：在设置页面点击"立即同步"按钮

同步范围：状态为"招聘中"的职位会自动发布到 Careers 页面。

路径：顶部导航 → **日程**

日程页面以日历/列表形式展示所有面试安排：
- 按日期分组显示
- 每条记录包含：候选人姓名、面试轮次、面试官、时间、地点

## 八、Offer 管理

路径：顶部导航 → **Offer**

### 8.1 Offer 列表
展示所有 Offer 记录：
- 候选人姓名、应聘岗位、Offer 状态、薪资信息、发放时间

### 8.2 创建 Offer
在候选人详情页或 Offer 管理页创建：

| 字段 | 说明 |
|------|------|
| 候选人 | 自动关联 |
| 岗位 | 自动关联 |
| 薪资 | 文本输入 |
| 入职时间 | 日期选择 |
| 备注 | 其他 Offer 条件说明 |

### 8.3 Offer 状态流转
```
待发放 → 已发放 → 已接受 / 已拒绝
                 ↘ 已撤回
```

## 九、猎头管理

路径：顶部导航 → **猎头**

### 9.1 猎头列表
管理合作猎头信息：
- 猎头名称、联系方式、所属公司
- API Key（用于猎头门户登录）
- 推荐候选人数量统计

### 9.2 新增猎头
创建猎头账号并生成 API Key，猎头可通过专属门户提交候选人。

### 9.3 猎头门户
猎头通过 API Key 登录独立门户页面：
- 查看可推荐的在招职位
- 提交候选人信息（姓名、联系方式、简历等）
- 查看已推荐候选人的状态进展

管理员可在猎头管理页查看各猎头的推荐记录和转化情况。

## 十、系统设置

路径：顶部导航 → **设置**（仅管理员可见）

### 10.1 来源管理
- 管理候选人来源渠道列表（如：Boss直聘、拉勾、猎头推荐、内推等）
- 点击来源标签旁的 **×** 可删除该来源
- 输入框中输入新来源后点击"添加"

### 10.2 标签管理
- 管理候选人自定义标签（如：高潜、紧急、待定、优秀、内推优先等）
- 点击标签旁的 **×** 可删除该标签
- 输入框中输入新标签后点击"添加"

### 10.3 用户管理
- 查看系统所有用户
- **新增用户**：设置用户名、密码、角色（admin / member）
- **编辑用户**：修改用户信息和角色
- **删除用户**：移除用户
- 飞书登录的用户会自动创建/关联

### 10.4 SMTP 邮件配置
- 配置邮件服务器信息（用于发送面试邀请邮件等）
- 配置项：SMTP 主机、端口、用户名、密码、发件人地址

### 10.5 Careers 同步配置
- 配置官网 Careers 页面同步相关参数
- 手动触发同步按钮

## 十一、飞书集成功能

系统与飞书深度集成，提供以下能力：

### 11.1 单点登录（SSO）
- 通过飞书 OAuth 实现扫码登录
- 自动关联飞书用户与系统用户

### 11.2 消息通知
以下事件会自动发送飞书通知：
- 面试安排通知（通知面试官）
- 面试评价提交通知（通知 HR）
- 候选人状态变更通知
- 审批流程通知

### 11.3 日历同步
- 安排面试时可同步创建飞书日历事件
- 面试官在飞书日历中可直接查看面试安排

### 11.4 任务同步
- 系统可自动在飞书中创建待办任务
- 支持设置任务到期时间

### 11.5 用户目录同步
- 可从飞书组织架构同步员工信息
- 用于面试官选择和用户匹配

## 十三、常见操作流程

### 13.1 完整招聘流程
```
1. 创建职位（职位管理 → 新增）
2. 录入候选人（手动新增 / Excel 导入 / 猎头推荐）
3. 筛选简历（看板：待筛选 → 简历初筛）
4. 安排面试（候选人详情 → 安排面试 → 选择面试官和时间）
5. 面试官评价（面试官收到通知 → 提交面评）
6. 推进/淘汰（根据面评结果推进到下一轮或淘汰）
7. 重复 4-6 直到面试流程完成
8. 发放 Offer（候选人详情 → 创建 Offer）
9. 跟踪 Offer 状态（已发放 → 已接受）
10. 标记入职
```

### 13.2 批量招聘场景
```
1. 准备 Excel 候选人名单
2. 候选人列表 → 导入 Excel
3. 在看板中批量筛选（使用快捷按钮推进/淘汰）
4. 为通过筛选的候选人逐一安排面试
```

### 13.3 猎头协作流程
```
1. 设置 → 猎头管理 → 新增猎头（生成 API Key）
2. 将 API Key 和门户地址发送给猎头
3. 猎头通过门户提交候选人
4. HR 在候选人列表中查看猎头推荐的候选人（来源：猎头推荐）
5. 正常进入招聘流程
```

## 十五、注意事项

1. **数据安全**：定期备份 data.json 文件，该文件存储了所有业务数据
2. **简历存储**：推荐配置 Supabase 云存储，确保简历文件不会丢失
3. **飞书配置**：需要在飞书开放平台创建应用并配置相关权限，才能使用飞书集成功能
4. **时区**：系统默认使用北京时间（Asia/Shanghai）
5. **浏览器兼容**：推荐使用 Chrome / Edge 等现代浏览器访问
